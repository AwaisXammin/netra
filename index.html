The Beauty of Simplicity

Simplicity is a virtue of the finest and most successful products—reducing a product to its core functions, but optimizing those to perfection. That’s what we did with the Netra platform.

The beauty of Netra lies in the simplicity of its architecture. It consists of a set of highly optimized modules that perform the heavy-lifting of IDP-related tasks. Yet, it is extremely easy to integrate into an existing IT landscape. The glue between these modules does not come in the form of a complicated user-interface layer as in many other IDP products, but in a very flexible layer of Python code. What looks like a simple idea at first, soon demonstrates its usefulness when it comes to interconnecting the IDP solution to existing data sources or hooking up to a large variety of services. This layer results in an unprecedented flexibility when creating new IDP models. And instead of asking your data specialists to learn Python and the functionality of our libraries, we will write those modules for you—if we don’t have them already on stock.

How it Works

Every IDP project is different, but there is a common blueprint that underlies all projects.
Starting an IDP project with Netra is especially easy.

Inquiry

You inquire about a model for a particular application. Either we have that model already in stock, in which case we continue with phase Model Evaluation or we will develop a tailor-made model for you. In that phase we also discuss where your operational data is sourced from and where the extracted data will be placed.

Training Set Preparation

You send us a set of sample documents that we call the training set. As preparation for the development of a model, this training set needs to be annotated, that is we manually determine the desired output data for each document. The training set should have a reasonable size, so the model we build is prepared for all those variations that occur in real-world documents.

Model Training

We will develop the new model and use automated training mechanisms to fine-tune the model accordingly. This step is a combination of software development and machine learning. The development part includes telling the model where to find the various pieces of information. The training will prepare the model for any kind of variations in format and placement of fields.

Model Evaluation

You send us a second set of sample documents, which again will be annotated. We call this the evaluation set. This set is used to verify that the model works as expected, indepen¬dently from the training set. The annotation allows an automated check of the model’s correctness.

Deployment

The model will be deployed at your premises and on your own hardware. Of course, we will support you during this phase to get things running as seamlessly and quickly as possible. We do not require your personnel to become IDP specialists or do weeks of training and preparations.

Maintenance

The world is in motion and so are your documents. Changes do occur over time. And those changes sometimes have to be reflected in the IDP software. We will stand by your side, should any changes become necessary, be it to the model or the embedding in your infrastructure.
